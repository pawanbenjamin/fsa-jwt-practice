This API contains two endpoints. One for ‚úçÔ∏è signing up and one for ü§ù authenticating a user.

  • /signup

    Must pass an object with username and password properties in request body.

    Returns an object with success, message and token properties

    Sample Call:

              
            fetch('https://fsa-jwt-practice.herokuapp.com/signup', 
              { 
                method: "POST", 
                headers: { 
                  "Content-Type": "application/json" 
                }, 
                body: JSON.stringify({ 
                  username: "some-username", 
                  password: "super-secret-999" 
                }) 
              })

    Sample Response:

              
            {
              "success": true,
              "message": "Thanks for signing up!",
              "token": "...ommitted for brevity"
            }

  • /authenticate

    The incoming request must send the token back in authorization header in following format:

    Authorization: `Bearer ${token}`

    On success, will return an object with success, message and data properties

    Sample Call:

              
            fetch('https://fsa-jwt-practice.herokuapp.com/authenticate', 
              { 
                method: "GET", 
                headers: { 
                  "Content-Type": "application/json",
                  "Authorization": `Bearer ${your-token-here}` 
                }
              })

    Sample Response:

              
            {
              "success": true,
              "message": "Correctly Authenticated!",
              "data": {
                "username": "Barry",
                "iat": 1684331768
              }
            }
